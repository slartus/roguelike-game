# Процедурные анимации и gif-документация

Процедурные анимации описаны в docs текстом и **сопровождаются gif-ом**, встроенным прямо в раздел сущности.

- gif'ы лежат в `docs/gamedesign/media/*.gif`.
- Генератор — `tools/gen_animation_gifs.py` (Python + Pillow).
- Скрипт повторяет `_draw` / tween-формулы соответствующих `.gd` **один-в-один** — так gif гарантированно отражает то, что видит игрок.

## Что считается «процедурной анимацией»

Три категории — правило действует на все три:

1. **`_draw`-render** — процедурная отрисовка через `draw_circle` / `draw_line` / `draw_polyline` в `_process` с `queue_redraw()`.
   Примеры: `poison_cloud.gd`, `spider_web.gd`.
2. **Твины scale / modulate / position** — покадровое мутирование поля `Sprite2D` / `Node2D` по формуле с временем/фазой.
   Примеры: `slime.gd::_apply_visual_bounce`, `lich.gd::_apply_cast_visual`, `boss.gd::_apply_cast_visual`.
3. **Sprite-frames / AnimatedSprite2D** — покадровая анимация из массива текстур (в проекте сейчас **не используется**; если появится, генерируем через прямую сборку из PNG-frame'ов).

## При изменении процедурной анимации

Любое изменение констант, формул, цветов, фаз, кеша геометрии, `_draw`-кода — обязательно:

1. Обновить соответствующую `render_<name>()` функцию в `tools/gen_animation_gifs.py` под новую логику.
2. Перегенерировать gif: `python3 tools/gen_animation_gifs.py` (или skill `/regen-gifs`). Скрипт перепишет только затронутый файл в `docs/gamedesign/media/`.
3. Открыть получившийся gif визуально — совпадает ли с ожиданиями (`open docs/gamedesign/media/<name>.gif`).
4. Закоммитить `.gd`-правку + правку `gen_animation_gifs.py` + новый `<name>.gif` **в тот же коммит**.

Разъезжание между кодом и gif — регрессия. Ревьюер обязан ловить.

## При добавлении новой процедурной анимации

1. Написать `render_<name>()` в `tools/gen_animation_gifs.py` и добавить запись в `ANIMATIONS`.
2. Сгенерировать gif.
3. Встроить в docs: `![<caption>](media/<name>.gif)` в соответствующий раздел `docs/gamedesign/*.md`, обычно рядом с описанием эффекта.
4. Всё это — в том же коммите, что и код фичи.
