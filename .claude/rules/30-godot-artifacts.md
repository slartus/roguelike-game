# Godot артефакты в git

Godot и генераторы (Pillow-скрипты, csv-translation importer, `.import`-файлы) создают ряд файлов, часть из которых **обязана быть в репо**, чтобы игра работала при свежем clone.

## Что коммитить

| Файл | Почему |
|------|--------|
| `resources/translations/*.translation` | Godot резолвит `tr()` по этим бинарным ресурсам; без них — сырые ключи в UI при первом запуске |
| `assets/**/*.png` + `*.png.import` | `.import` хранит `uid` + импорт-настройки (filter, mipmaps); удаление ломает ссылки в `.tscn` |
| `resources/weapons/*.tres` + `*.tres.import` | Godot Resource + import metadata |
| Скрипты-генераторы `tools/gen_*.py` | Детерминированные; коммитим **и** генератор, **и** его output |
| `docs/gamedesign/media/*.gif` | Часть game-design docs, генерируются `tools/gen_animation_gifs.py` |
| `*.gd.uid`, `*.tres.uid`, `*.tscn.uid` | Godot 4.4+ хранит стабильные `uid://` ссылки; без `.uid` рядом со скриптом — Godot регенерирует и меняет id, ломая `preload("uid://…")` в других сценах |

## Что НЕ коммитить

| Путь | Почему |
|------|--------|
| `.godot/imported/**` | Кэш импорта, регенерируется при первом старте |
| `.godot/uid_cache.bin` | Регенерируется |
| `.import/` (legacy) | То же самое для Godot 3 legacy |
| `dist/` | Экспортированные билды |
| `export.cfg`, `export_presets.cfg` содержит секретные ключи? — коммитить `.cfg` можно, но не приватные ключи |

## Правило

Если файл нужен Godot для запуска игры и **не регенерируется автоматически** при первом старте (не в `.godot/`) — коммитить. При обновлении `.gitignore` — прогони:

```bash
git check-ignore -v <path>
```

для критичных файлов и убедись, что они не заблокированы.

## Ловушка `.gd.uid`

Если у файла `.gd` рядом нет `.gd.uid` — Godot при первом открытии редактора сгенерирует новый `uid://` для этого скрипта, и все `preload("uid://old_id")` в других сценах сломаются. Всегда коммитим `.gd.uid` рядом с `.gd`. Это же касается `.tres.uid`, `.tscn.uid`.
