# Game Design Docs

Живая документация геймплейных сущностей игры. Каждый файл описывает **текущее** состояние: параметры, поведение, взаимодействия, пути к сценам/скриптам.

## Правило

При **любом** изменении геймплейных сущностей — параметров, поведения, добавления или удаления — соответствующий файл обновляется в **том же коммите**. См. `/CLAUDE.md` → раздел «Game Design Docs».

## Что НЕ описывать здесь

- Структура кода / архитектурные решения — они в самом коде.
- Реализация конкретного скрипта на GDScript — описываем **что**, а не **как**.
- Задачи на будущее — только текущее состояние. Планы — в issue или отдельных ADR.
