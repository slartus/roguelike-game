# Workflow фичи

Одно сообщение пользователя = одна фича = один коммит = один push.

## 6 обязательных шагов перед считать фичу сданной

1. **Docs** — обновить `docs/gamedesign/*.md` если задета game-design сущность (см. `20-gamedesign-docs.md`).
2. **Tests** — написать/обновить тесты покрывающие изменённое поведение (см. `10-tests.md`).
3. **Run tests** — прогнать всю GUT-сюиту через CLI. Красный тест = не коммитить.
4. **Self-review** — прогнать `godot-code-reviewer` через Task tool (см. `.claude/agents/godot-code-reviewer.md`). `BLOCKER/CRITICAL` → фикс → новый проход → тесты снова.
5. **Commit** — отдельный коммит на фичу, не смешанный с другими.
6. **Push** — `git push`. Без push фича не сдана.

Быстрый запуск всего цикла: skill `/feature-close`.

## Правило одного сообщения

Каждый пользовательский запрос — **новая фича**, даже если тематически похожа на предыдущую. Отдельный коммит, отдельный push.

**Исключение — только явные слова пользователя:** «это правка к текущей», «доработай текущую», «в тот же коммит», «в текущую фичу», «amend». Иначе — новая фича.

Если во время выполнения одной фичи пришло новое сообщение — не бросай текущую, добавь в очередь через `TaskCreate`, доведи текущую до push'а, потом бери новую.

Если один запрос физически требует несколько зависящих коммитов (миграция → refactor → cleanup) — каждый под-коммит проходит все 6 шагов и пушится отдельно.

## Освобождения

Только `docs/*.md` / `CLAUDE.md` / `.claude/**` изменены и `.gd`/`.tscn`/`.tres` не тронуты → шаги 2 и 3 можно пропустить. Шаги 1, 4, 5, 6 остаются.

## Команда прогона тестов

```bash
/Applications/Godot.app/Contents/MacOS/Godot --headless --path . \
  -s addons/gut/gut_cmdln.gd -gdir=res://test/unit -gexit
```

Или через skill `/run-tests`.

Если тесты падают не из-за твоих правок (инфра, сторонний баг GUT) — явно пишешь это в commit message, не маскируешь.

## Push

`git push` разрешён по умолчанию (`.claude/settings.local.json`). Не запрашиваю подтверждения.

Останавливаюсь и уточняю только в двух случаях:
1. `git push --force` / `--force-with-lease` в `main` — деструктивно, нужно согласие.
2. Push отклонён remote'ом (protected branch, конфликт) — разбираться, не делать force «на автомате».

Проверяю ответ push'а — `To ... main -> main` или `Everything up-to-date`. Пока не увидел успех — фича не завершена.
