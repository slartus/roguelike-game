# Локализация

Все пользовательские строки в игре берутся через `tr(KEY)` из CSV-таблицы переводов.

## Файлы

- `resources/translations/strings.csv` — источник. Первая колонка `keys`, дальше — коды локалей.
- `resources/translations/strings.csv.import` — importer settings (Godot `csv_translation`, `delimiter=0` — запятая).
- `resources/translations/strings.en.translation` — генерируется автоматически при импорте CSV.
- `resources/translations/strings.ru.translation` — то же.

`project.godot` регистрирует эти `.translation` файлы через `[internationalization] locale/translations`. Fallback locale — `en`.

## Активная локаль

`EventLog._ready` вызывает `TranslationServer.set_locale("ru")` — стартовая локаль русская. Смена локали:

```gdscript
TranslationServer.set_locale("en")  # или "ru"
```

Все `tr()` вызовы сразу подхватят новый язык — но уже отрисованные Label в HUD не обновятся автоматически, их нужно перерисовать через `set_*` методы.

## Категории ключей

| Префикс | Что | Где |
|---------|-----|-----|
| `UI_*` | Строки HUD-панели | `hud.gd::set_*` через `tr()` |
| `LOG_*` | Шаблоны combat log | `event_log.gd::log_*` через `tr()` |
| `ENEMY_*` | Имена врагов | `.tscn::display_name` (ключ), `EventLog.log_kill` резолвит |
| `WEAPON_*` | Имена оружия | `.tres::display_name` (ключ), `EventLog.log_weapon_pickup` резолвит |

## Как добавить строку

1. Добавь строку в `resources/translations/strings.csv`: `KEY,"english text","русский текст"`.
2. Запусти Godot один раз (или `godot --headless --editor --quit-after 30`) чтобы регенерировать `.translation` файлы.
3. Используй ключ в коде: `tr("KEY")` или `tr("KEY") % args`.
4. Закоммить `strings.csv` **и** оба `.translation` файла (они бинарные, но нужны для запуска без импорта).

## Как добавить локаль

1. Добавь колонку в `strings.csv` (например `de`).
2. Обнови `project.godot`: добавь `res://resources/translations/strings.de.translation` в `locale/translations`.
3. Godot сгенерирует файл при импорте.

## Правила именования ключей

- `UPPER_SNAKE_CASE`, без пробелов и кириллицы.
- Префикс по категории: `UI_`, `LOG_`, `ENEMY_`, `WEAPON_`, `HINT_`, `DIALOG_`.
- Не описывать значение — описывать назначение. Плохо: `WEAPON_YELLOW_KILLER`. Хорошо: `WEAPON_PISTOL`.

## Что тестируется

`test/unit/test_event_log.gd`:
- В `before_each` фиксирует локаль `ru`, восстанавливает в `after_each`.
- Проверяет русские переводы для всех `log_*` методов.
- Отдельный тест `test_english_locale_translates_correctly` проверяет что `en` тоже работает.

При добавлении новой строки — покрой её тестом на обеих локалях, если она попадает в combat log или UI.
